    Methods

    # init(width, height, options)

    This function creates the canvas and webgl canvas context and initializes the renderer and included event handlers.
    Parameters:
    Name Type Default Description
    width number 100 The width of the created canvas.
    height number 100 The height of the created canvas.
    options Object An object with options for initialization.
    Properties
    Name Type Default Description
    noUpdate boolean false Stops the built in update function from immediately running if true.
    canvas HTMLCanvasElement Supply a canvas to which Hummingbird must hook, instead of creating a new one.
    parent HTMLElement document.body Supply a parent element to create our canvas element under.
    id string HummingbirdCanvas The ID of the canvas in the DOM.

    # resizeCanvas(width, height)

    This function resizes the canvas element and updates this in the rendering backend.
    Parameters:
    Name Type Default Description
    width number 100 The width to resize the canvas to.
    height number 100 The height to resize the canvas to.

    # setup()

    The main setup function of Hummingbird, initializes in-engine textures and math objects, gets called on the window 'load' event. When this finishes, the global setup function is called.
    Fires:
    • global:setup

    # update()

    The main update function of Hummingbird, updates the camera and starts a render batch. Calls update, which runs at the vsync rate of the display (for drawing) and fixedUpdate, which runs at external:HB#fixedUpdateFrequency (suitable for physics).
    Fires:
    • global:update
    • global:fixedUpdate
